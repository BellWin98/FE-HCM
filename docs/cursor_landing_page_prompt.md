## Role
너는 10년 경력의 시니어 프론트엔드 개발자이자 CRO(Conversion Rate Optimization) 전문가야.
랜딩페이지 설계, UX 카피라이팅, 전환율 최적화에 깊은 전문성을 갖고 있어.

## Context
현재 랜딩페이지가 존재하지만 전환율이 낮아서 전면 개선이 필요해.

## Task
다음 순서대로 진행해줘.

---

### Phase 1: 프로젝트 전체 분석
먼저 프로젝트 전체를 분석해서 핵심 기능을 파악하고, 아래 항목을 파악한 뒤 요약 보고를 해줘.
실제 코드를 수정하기 전에 반드시 이 분석 결과를 먼저 보여줘.

1. **현재 랜딩페이지 구성**: 어떤 섹션이 있고, 어떤 순서로 배치되어 있는지
2. **기존 디자인 시스템**: 색상 팔레트, 타이포그래피, 공통 컴포넌트, 디자인 토큰
3. **현재 문제점 진단**: 랜딩페이지 관점에서의 구조적/UX적 문제점 리스트

---

### Phase 2: 랜딩페이지 구조 개선
분석 결과를 바탕으로, 아래 Best Practice 섹션 구조에 맞게 랜딩페이지를 재설계해줘.
기존 디자인 시스템(색상, 폰트, 컴포넌트)을 최대한 유지하면서 개선해.

#### 필수 섹션 구조 (이 순서를 따를 것)

1. **Hero Section**
   - 고객의 핵심 문제/욕구를 해결하는 헤드라인 (혜택 중심, "~됩니다" 형태)
   - 서비스가 어떻게 문제를 해결하는지 서브헤드라인 1~2줄
   - 명확한 Primary CTA 버튼 (행동 동사 + 혜택: "무료로 시작하기")
   - 안심 문구 (CTA 아래에 기재)
   - 서비스 스크린샷 또는 데모 영상 영역 (내가 서비스 데모영상을 GIF파일로 줄거야. 유저들이 한눈에 볼 수 있는 공간만 마련해)

2. **Social Proof Bar (신뢰 로고)**
   - 유저는 10명 밖에 없지만 매일 만족하며 사용하고 있다는 문구 
   - 핵심 숫자 지표 (사용자 수, 만족도 등)

3. **핵심 혜택 섹션 (Benefits, NOT Features)**
   - 3~4개 핵심 혜택을 아이콘 + 제목 + 설명 카드로 구성
   - 기능이 아닌 혜택 중심 카피 ("AI 엔진 탑재" ❌ → "매일 3시간 절약" ✅)

4. **작동 방식 (How It Works)**
   - 3단계로 단순화 (가입 → 설정 → 결과 확인)
   - 각 단계에 번호 + 아이콘 + 짧은 설명

5. **상세 기능 쇼케이스**
   - 주요 기능 2~3개를 이미지/GIF + 텍스트 교차 레이아웃으로 배치
   - 각 기능마다 구체적 숫자 포함 ("평균 2.3초 내 처리")

6. **고객 후기 / 사례 (Testimonials)**
   - 후기 카드: 인용문 + 이름 + 직함 + 회사 + 사진 placeholder
   - 캐러셀 또는 그리드 레이아웃

7. **요금제 섹션 (Pricing) - 해당되는 경우**
   - 2~3개 플랜 비교 카드
   - 추천 플랜 하이라이트
   - 각 플랜에 CTA 버튼

8. **FAQ 섹션**
   - 아코디언 형태로 5~7개 질문
   - 자주 묻는 전환 장벽 해소 질문 위주

9. **최종 CTA 섹션 (Bottom CTA)**
   - 행동을 유도하는 마무리 헤드라인
   - Hero와 동일한 CTA 버튼 반복
   - 긴급성 요소 (한정 혜택 등) 선택적 추가

10. **Footer**
    - 필수 링크 (개인정보처리방침, 이용약관, 연락처)
    - 소셜 미디어 링크

### Phase 3: UX/UI 최적화 규칙

아래 규칙을 반드시 적용해줘.

#### 전환율 최적화
- [ ] 페이지 내 네비게이션 링크 최소화 (외부 이탈 방지)
- [ ] CTA 버튼은 한 가지 목표에만 집중 (회원가입 OR 데모 신청)
- [ ] CTA 버튼 색상은 페이지 내 다른 요소와 확실히 대비
- [ ] 스크롤 시 상단 고정 Sticky CTA 또는 Floating CTA 구현
- [ ] CTA 버튼 주변에 안심 문구 배치

#### 카피라이팅
- [ ] 모든 텍스트는 한국어로 작성 (placeholder 포함)
- [ ] "우리 서비스는~" ❌ → "당신은~", "여러분의~" ✅ (고객 중심)
- [ ] 구체적 숫자 사용 ("빠른" ❌ → "2.3초" ✅)
- [ ] 전문 용어 최소화, 쉬운 언어로 작성

#### 반응형 & 모바일
- [ ] 모바일 퍼스트 설계 (breakpoint: 640px / 768px / 1024px / 1280px)
- [ ] 버튼 터치 영역 최소 48px
- [ ] 모바일에서 텍스트 최소 16px
- [ ] 긴 콘텐츠는 아코디언/탭으로 접기
- [ ] 모바일에서 CTA 버튼이 항상 화면 하단에 고정되도록

#### 퍼포먼스
- [ ] 이미지에 lazy loading 적용
- [ ] 적절한 시맨틱 HTML 태그 사용 (section, article, nav, etc.)
- [ ] SEO 메타 태그 최적화 (title, description, og 태그)
- [ ] 스크롤 애니메이션은 가벼운 CSS transition 또는 Intersection Observer 활용
- [ ] 불필요한 외부 라이브러리 추가 금지 (기존 스택 내에서 해결)

#### 접근성
- [ ] 모든 이미지에 alt 텍스트
- [ ] 충분한 색상 대비 (WCAG AA 기준 4.5:1 이상)
- [ ] 키보드 네비게이션 지원
- [ ] aria-label 적절히 사용

### Phase 4: 인터랙션 & 마이크로 애니메이션

아래의 인터랙션을 적절히 추가해줘 (과하지 않게).

- 스크롤 시 섹션별 fade-in / slide-up 등장 효과
- CTA 버튼 hover 시 미세한 scale + shadow 변화
- 숫자 지표에 카운트업 애니메이션
- FAQ 아코디언 부드러운 열림/닫힘 transition
- 고객 후기 자동 슬라이드 (선택)

## Output Format

## Constraints (반드시 지킬 것)

1. 기존 프로젝트의 기술 스택과 디자인 시스템을 유지할 것
