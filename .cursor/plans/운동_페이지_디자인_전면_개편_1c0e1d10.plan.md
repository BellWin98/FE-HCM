# 운동 관련 페이지 디자인 전면 개편 계획

## 현재 상태 요약

- **색상**: 전역적으로 `from-blue-50 to-indigo-100` 배경, `blue-600` 로고/CTA, `bg-gray-50` 레이아웃. MyPage만 `from-purple-600 to-pink-600`로 이탈.
- **구성**: 카드+그라데이션 히어로, 얌전한 버튼/링크. 모바일은 기본 반응형만 적용.
- **영향 범위**: [Index.tsx](src/pages/Index.tsx), [LoginPage.tsx](src/pages/LoginPage.tsx), [RegisterPage.tsx](src/pages/RegisterPage.tsx), [DashboardPage.tsx](src/pages/DashboardPage.tsx), [WorkoutUploadPage.tsx](src/pages/WorkoutUploadPage.tsx), [CreateRoomPage.tsx](src/pages/CreateRoomPage.tsx), [MyPage.tsx](src/pages/MyPage.tsx) 및 이들이 사용하는 Layout/Header, 대시보드·마이페이지 하위 컴포넌트. **주식 관련 페이지·컴포넌트는 제외.**

## 1. 디자인 방향 및 브랜드 정의

### 1-1. 타겟·컨셉·참고 서비스

| 항목 | 정의 |
|------|------|
| **타겟** | 운동을 즐기는 2030 MZ세대 |
| **컨셉** | 스포츠, 게이미피케이션, 다크 모드. 나이키·스트라바·다크 게임 UI처럼 강렬하고 스포티한 느낌 |
| **참고** | **Strava** — 다크+오렌지 포인트, 기록·데이터. **Nike Run Club** — 블랙+네온 액센트, 굵은 타입·모션. **Duolingo** — 스트릭·XP 게이미피케이션, 달성감·재미 |

### 1-2. 컬러 팔레트 (Hex 4종)

| 역할 | 이름 | Hex | 용도 |
|------|------|-----|------|
| 1 | Deep Black | #0A0A0A | 전역 배경, 풀블리드. 가독·몰입 균형 |
| 2 | Dark Surface | #141414 | 카드·헤더·입력 필드. 계층 구분 |
| 3 | Neon Lime | #B8FF3C | CTA, 강조 텍스트, 아이콘, 프로그레스·배지 |
| 4 | Off-White | #F2F2F2 | 본문·제목. 다크 위 대비, 눈 피로 완화 |

- **확장**: 벌금·경고 **Alert Red #EF4444** (5번째). 오렌지 대안 **Intense Orange #FF6B00**.

### 1-3. 디자인 무드 (Mood)

- **한 줄**: "다크 위에서 네온 라임이 터지는, 스포츠 × 게이미피케이션 랜딩."
- **Bold & High-Contrast**: 깊은 다크 배경, 선명한 텍스트·CTA. Strava/Nike처럼 정보와 행동이 즉시 읽히게.
- **Sporty & Energetic**: Neon Lime으로 "움직임·기록·도전" 전달. 큰 타이포·숫자·강한 CTA로 에너지 집중.
- **Gamified & Achievable**: Duolingo처럼 목표·스트릭·벌금이 숫자/뱃지/미니 차트로 보이게 해 "해보면 달성할 수 있다"는 느낌.
- **Dark-First & Immersive**: 화면 전체 딥 다크, 앱/게임처럼 몰입, 모바일 한 화면 집중.
- **Trust & Clarity**: "무엇을 하는 서비스인지 → 왜 가입해야 하는지 → 지금 시작"이 3초 안에 읽히게.

### 1-4. 컨셉 키워드 반영

| 키워드 | 반영 방향 |
|--------|-----------|
| **운동** | 역동성, 힘, 에너지 — 강한 대비, 굵은 타이포, 미세한 모션(호버/등장) |
| **경쟁** | 긴장감, 순위, 승부 — Neon Lime·Alert Red 포인트, 랭킹/벌금 강조 UI |
| **벌금** | 리스크, 경고 — Alert Red 일관, 벌금 숫자·라벨 시각적 강조 |

### 1-5. 타이포·모션·토큰 적용

- **타이포**: 히어로/제목 `font-bold` ~ `font-black`, 자간/행간 조정.
- **모션**: `tailwindcss-animate` — 버튼/카드 `hover:scale-[1.02]`, 섹션 `animate-in fade-in slide-in`.
- **모바일**: 랜딩·로그인 full-bleed 배경, 터치 영역 44px 이상.
- **CSS/테마**: [src/index.css](src/index.css) `:root` 에 `--brand-bg`(#0A0A0A), `--brand-surface`(#141414), `--brand-primary`(#B8FF3C), `--brand-foreground`(#F2F2F2), `--brand-penalty`(#EF4444) 추가. [tailwind.config.ts](tailwind.config.ts) `theme.extend.colors` 에 브랜드 색 확장. 운동 플로우만 적용, 주식·기존 primary 와 충돌 최소화.

## 2. 공통 컴포넌트 (운동 플로우 일관성)

### 2-1. Layout & Header

- [Layout.tsx](src/components/layout/Layout.tsx): `bg-gray-50` → `--brand-bg`(Deep Black #0A0A0A) 적용. 운동 페이지 전용일 때만 적용하도록 variant(`default` / `workout` 등) 고려.

## 3. 페이지별 개편 포인트

### 3-1. Index (랜딩) — 후킹 목표

**목표**: 처음 방문한 사용자가 **무슨 서비스인지 단번에 파악**하고, **쓰고 싶어서 가입까지 이어지도록** 후킹. 단순 디자인 수정이 아니라 **정보 구조·콘텐츠 배치**까지 포함.

- **Above the fold (첫 화면)**  
  - 배경: **Deep Black #0A0A0A** 풀블리드. 그라데/미세 노이즈 선택적 적용.  
  - 히어로 문구: "돈을 잃어야 움직인다." — **크기·굵기 강화**(예: `text-4xl sm:text-6xl font-black`), **Neon Lime #B8FF3C** 그라데이션 또는 단색.  
  - 한 줄 설명: "친구와 방 만들고, 운동 인증하고, 못 하면 벌금" 같은 **핵심 가치 제안 1문장**으로 3초 안에 전달. Off-White #F2F2F2.  
  - CTA: "무료로 시작하기" — **Neon Lime** 배경, 다크 텍스트, 호버 시 scale/밝기.  
  - 모바일: 히어로 `min-h-[70vh]` 이상, CTA 하단 고정 또는 스크롤 후 등장으로 몰입감·전환 유도.

- **핵심 기능 노출 (스크롤)**  
  - **친구와 함께 / 목표·인증 / 벌금 / 성취감** — 4개 카드를 **Dark Surface #141414** 카드 + 보더(라임 얇게 또는 neutral). 아이콘은 Neon Lime·Alert Red 등으로 키워드별 구분.  
  - **3단계(방 생성 → 운동 인증 → 함께 성장)** — `bg-[#141414]` 섹션, 넘버링을 Neon Lime으로.  
  - 게이미피케이션 느낌: "주 N회", "벌금 N원", "스트릭" 같은 **숫자·뱃지**를 Duolingo/Strava처럼 눈에 띄게.

- **디자인 톤**  
  - 배경: `from-blue-50 to-indigo-100` 제거 → **#0A0A0A** 기준.  
  - 카드: 흰 카드 제거 → **#141414** 또는 반투명 다크 + backdrop-blur.  
  - 링크/버튼: `text-blue-600` 제거 → **Neon Lime** 또는 Off-White.

## 4. 모바일 몰입감 강화

- **Index/Login/Register**: full-viewport #0A0A0A, 히어로/폼에 `pb-safe`·safe-area. CTA `fixed bottom` + safe-area padding 선택.
- **Dashboard**: 스크롤 일관화, 카드 간격·터치 타겟 44px 이상, 탭 패딩 확대.
- **WorkoutUpload/CreateRoom**: 폼 블록·입력 필드 높이 확대, 제출 버튼 하단 고정 또는 스크롤 후 항상 노출.
- **전역**: [index.css](src/index.css) `.pb-safe`, `.pb-safe-bottom` 유지, `min-h-dvh` 활용.

## 5. 작업 순서 제안

1. **디자인 토큰**: [index.css](src/index.css) + [tailwind.config.ts](tailwind.config.ts) 에 브랜드 Hex 4종·Alert Red 변수 추가, 운동 플로우용 클래스/변수 매핑.
2. **Layout & Header**: 배경 Deep Black, 로고 Neon Lime, 운동/주식 variant 고려.
3. **Index** (랜딩): 후킹 목표 반영 — 히어로·핵심 기능·3단계·CTA 재구성 + Neon Lime·Deep Dark 적용.
4. **Login** → **Register**: 다크 톤·카드·링크 통일.
5. **Dashboard + 하위**: DashboardHeader, StatsCards, MyActivityCard, MyWorkoutRoom, MemberStatus, AvailableWorkoutRooms, ChatRoom, Penalty, 탭.
6. **WorkoutUploadPage + WorkoutSuccessDialog** → **CreateRoomPage**.
7. **MyPage + UserProfileSection, WorkoutStatsSection, WorkoutFeedSection**.
8. **로딩/스피너**: App, RequireRole, OAuthCallback 등에서 blue → Neon Lime(primary).

## 6. 주의사항

- **주식 관련**([StockPortfolioPage](src/pages/StockPortfolioPage.tsx), [StockChart](src/components/StockChart.tsx), [StockAssetsTab](src/components/stock/StockAssetsTab.tsx), [StockProfitTab](src/components/stock/StockProfitTab.tsx), [StockHoldingListItem](src/components/stock/StockHoldingListItem.tsx), [TradingProfitLoss](src/components/TradingProfitLoss.tsx)): 손익 표시용 `text-blue-600` 등 **수정하지 않음**. 기존 의미(손실=파란 등) 유지.
- **Admin 페이지**: 요청 범위 외. 동일 Layout/Header 사용 시 색만 토큰으로 바뀌면 일관성 유지.
- **다크 모드**: `.dark` 변수 있으므로 새 토큰 정의 시 light/dark 둘 다 지정하면 테마 전환 시 유리.

이 순서대로 진행하면 **Strava·Nike·Duolingo**를 참고한 스포츠·게이미피케이션·다크 톤과 **Neon Lime + Deep Dark** 팔레트로, 랜딩에서 가입까지 이어지는 후킹과 모바일 몰입감을 확보하면서, 주식 페이지는 기존 디자인을 유지할 수 있습니다.
